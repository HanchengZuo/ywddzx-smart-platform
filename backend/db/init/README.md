# 数据库初始化脚本命名规则

PostgreSQL Docker 镜像会按照文件名字典序执行 `/docker-entrypoint-initdb.d` 下的 SQL 文件。

当前命名采用“阶段号 + 业务语义”的方式，而不是连续流水号：

- `10-19`：核心主数据，例如站点、用户、权限。
- `20-29`：巡检业务基础表，例如检查表注册、巡检记录、问题、批次。
- `30-39`：检查表规范库明细表结构。
- `40-49`：初始化种子数据。
- `50-59`：巡检计划相关表。
- `60-69`：证照等扩展业务表。

新增脚本时优先放入对应阶段区间，避免为了插入一个文件而重排全部初始化顺序。
