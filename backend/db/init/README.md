# 数据库初始化脚本命名规则

PostgreSQL Docker 镜像会按照文件名字典序执行 `/docker-entrypoint-initdb.d` 下的 SQL 文件。

当前命名采用“三位数阶段号 + 业务语义”的方式，而不是连续流水号。

- `100-199`：核心主数据与账号权限，例如站点、用户、权限覆盖表。
- `200-299`：巡检业务基础表，例如检查表注册、检查表字段、巡检记录、问题、批次。
- `300-399`：预留给巡检扩展对象；具体检查表明细表由管理页面动态创建，不再通过初始化脚本写死。
- `400-499`：预留给一次性兼容迁移或少量初始化数据；当前不内置任何检查表种子数据。
- `500-599`：巡检计划相关表，包含计划主表、计划站点明细，以及后续计划统计、计划日志等扩展表。
- `600-699`：证照等站点扩展业务表。
- `700-899`：预留给后续独立业务子系统。
- `900-999`：预留给紧急兼容脚本或部署期人工维护脚本。

新增脚本时优先放入对应三位数阶段区间，并在区间内按依赖顺序递增编号，避免为了插入一个文件而重排全部初始化顺序。
